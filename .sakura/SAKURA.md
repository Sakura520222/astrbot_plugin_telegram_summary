# 项目概述文档

## 项目简介
这是一个适用于 AstrBot 平台的 Telegram 频道消息总结插件，能够每周自动生成指定 Telegram 频道的消息汇总报告，并通过 AI 提取核心要点，自动推送到 QQ 群组或用户。

## 技术栈
- **主要语言**：Python
- **核心框架与库**：
  - Telethon：用于连接 Telegram API，抓取频道消息
  - APScheduler：用于定时任务调度（每周一早上 9 点自动生成周报）
  - OpenAI（或兼容 API）：用于 AI 驱动的消息总结和要点提取
  - AstrBot 插件系统：基于 AstrBot 框架的插件开发
  - python-dotenv：环境变量管理
- **配置管理**：通过 AstrBot 的配置系统进行参数设置，支持 YAML 格式

## 项目结构
项目目录结构简洁，核心文件如下：
- **main.py**：插件主入口，负责初始化、命令处理、定时任务和总结逻辑
- **metadata.yaml**：插件元数据，定义插件名称、版本、依赖等
- **requirements.txt**：列出项目依赖的 Python 库
- **_conf_schema.json**：配置项的 JSON 模式定义，用于 AstrBot 配置系统
- **README.md**：项目说明文档，包含安装、配置和使用指南
- **CHANGELOG.md**：版本更新日志
- **logo.png**：插件标识图标
- **.gitignore**：Git 忽略文件配置
- **LICENSE**：项目许可证文件

核心模块功能：
- **定时任务模块**：使用 APScheduler 调度每周总结任务
- **消息抓取模块**：通过 Telethon 从 Telegram 频道获取消息
- **AI 总结模块**：调用 AI 模型生成消息总结报告
- **推送模块**：将总结报告推送到 QQ 群组或用户
- **配置管理模块**：处理 AstrBot 配置系统中的参数设置
- **登录流程模块**：支持通过 QQ 聊天界面完成 Telegram 账号交互式登录

## 开发约定
从代码结构和功能推断的开发规范：
- **插件化设计**：遵循 AstrBot 插件开发规范，通过 main.py 和 metadata.yaml 集成到平台
- **配置驱动**：所有参数（如频道列表、AI 提示词、推送目标）通过 AstrBot 配置系统管理，避免硬编码
- **异步与定时任务**：使用 APScheduler 实现定时任务，确保每周自动执行总结
- **会话管理**：Telegram 登录采用会话控制，保存 session 文件以支持重启后自动加载
- **权限控制**：敏感操作（如登录、配置修改）仅限管理员通过 QQ 命令触发
- **错误处理与日志**：插件应包含基本的错误处理和日志记录，确保稳定运行
- **依赖最小化**：仅引入必要库（如 Telethon、OpenAI、APScheduler），保持轻量级
- **文档完整性**：提供详细的 README 和命令列表，便于用户理解和使用
